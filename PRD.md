## 1. 背景与目标

### 1.1 背景

对现有项目进行彻底改造：清理冗余代码，交付一款聚焦「韦林英文词根」学习的「轰炸」式练习产品。首版仅开放第一单元，其余单元以锁定状态呈现，为后续迭代预留结构。

### 1.2 产品目标

- 提供清晰、炫酷的单页学习入口，默认展示 30 个词根单元卡片。
- 首版完成第一单元可交互路径与「开始轰炸」主操作，其余单元仅展示锁定态。
- 技术栈统一为现代前端方案，便于后续扩展内容与动效。

### 1.3 成功标准（可验收）

- 页面上下分区布局符合规格：上区占主要高度，下区固定高度且按钮垂直居中。
- 30 张卡片正确渲染：1 张可用、29 张带锁定标识。
- 主题为黑灰系，视觉风格接近海外主流炫酷站点（高对比、克制霓虹或渐变可选）。
- 工程可 `vite` 构建运行，依赖与 `.gitignore` 配置合理；图标使用 Font Awesome。

---

## 2. 用户与场景

### 2.1 目标用户

- 正在使用韦林体系记忆英文词根的学习者。
- 需要短时、高强度（「轰炸」）重复刺激辅助记忆的用户。

### 2.2 核心场景

1. 打开应用 → 浏览 30 个单元 → 识别当前仅第一单元可进入。
2. 点击「开始轰炸」→ 进入第一单元对应的轰炸流程（具体交互可在实现阶段与产品对齐；本 PRD 以需求文件为准，首版以按钮与布局为主）。

> **说明**：若「开始轰炸」需跳转子页或弹层，建议在技术方案中单独列出路由/状态设计；需求原文以下半区单一按钮为主，本 PRD 将其定义为主行动点（Primary CTA）。

## 3. 功能需求

### 3.1 页面结构

| 区域 | 高度/布局 | 内容 |
|------|-----------|------|
| 上区 | 占据视口剩余主要高度（flex 或 grid 实现，`flex-1` / `min-h-0` 等避免溢出） | 词根单元卡片网格或列表，共 30 个 |
| 下区 | **固定高度**（建议与设计 token 一致，如 `h-20`～`h-24` 或明确 px 值） | 「开始轰炸」按钮，**垂直居中** |

### 3.2 卡片与单元状态

| 单元 | 数量 | 状态与表现 |
|------|------|------------|
| 第 1 单元 | 1 | 可点击/可进入（具体导航或模态在实现时定义，需与「开始轰炸」逻辑一致） |
| 第 2～30 单元 | 29 | **锁定**：展示锁定图标或角标，不可进入轰炸流程 |

**卡片信息（建议最小字段）**

- 单元序号或标题（如「Unit 1」～「Unit 30」）。
- 锁定态：Font Awesome 锁类图标 + 降低透明度或禁用 hover 反馈。

### 3.3 主操作

- **按钮文案**：「开始轰炸」
- **位置**：下区固定栏内水平与垂直均居中（相对下区容器）。
- **行为**：与第一单元可用态联动（例如仅当选中 Unit 1 或默认 Unit 1 时可用）；若需求简化为一键开始，则直接触发第一单元轰炸流。**首版以实现需求文件为准：以下区单一按钮为主。**

### 3.4 工程与基础设施

| 项 | 要求 |
|----|------|
| 技术栈 | React、TypeScript、Tailwind CSS、Vite、Framer Motion、axios |
| 图标 | Font Awesome（与 React 集成方式需在技术方案中选定：如 `@fortawesome/react-fontawesome`） |
| 版本控制 | **新增** `.gitignore`，包含 Node、Vite、IDE、OS 等常见忽略项 |
| 代码质量 | 删除历史冗余代码；目录与命名与新技术栈一致 |

## 4. 非功能需求

### 4.1 视觉与体验

- **主题**：默认黑灰系列（背景深、文字与边框层次清晰）。
- **风格**：参考海外主流炫酷站点（如精致渐变、微光边框、细腻阴影、可选 Framer Motion 入场/悬停动效），避免廉价高饱和堆砌。
- **响应式**：至少支持常见桌面宽度；移动端可为后续迭代（若首版需移动优先，应补充断点要求）。

### 4.2 性能与可维护性

- 首屏以静态结构 + 轻量动效为主，避免阻塞主线程的过重动画。
- TypeScript 严格模式建议开启；公共类型与单元数据结构集中定义，便于扩展 29 个解锁单元。

### 4.3 安全与合规

- 若后续 axios 对接 API，需在 PRD 附录中补充域名、鉴权与错误码策略；首版可无后端，axios 可预置实例与拦截器占位。

---

## 5. 范围外（本版不做）

- 第 2～30 单元的学习内容与解锁规则（仅 UI 锁定）。
- 详细的「轰炸」题目交互、进度、统计报表（除非在实现阶段与产品同步写入本文档）。
- 多语言界面（默认中文 UI 文案即可，与「英文词根」学习内容不冲突）。

## 6. 里程碑建议

| 阶段 | 交付物 |

| M1 | 工程脚手架（Vite + React + TS + Tailwind）、`.gitignore`、Font Awesome 接入 |

| M3 | 「开始轰炸」按钮与 Framer Motion 基础动效 + 主题定稿 |
| M4 | 清理旧代码、联调与视觉走查 |

## 7. 风险与假设

- **假设**：「开始轰炸」首版仅需触发占位流程或简单导航，复杂题库另版规划。
- **风险**：Framer Motion 与 Tailwind 动画职责重叠 —— 建议动效规范：布局过渡用 Framer，简单 hover 用 Tailwind。
- **风险**：30 卡片在窄屏下的网格折行 —— 需在 UI 稿或实现中明确列数与最小卡片宽度。
